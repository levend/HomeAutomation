[ 
    entities: account - homes - hubs - devices - sensor_values
]

-------- authentication
GET /login - authenticates the user, returns a temporary token
GET /logout - invalidates the token for a user
POST /accounts - creates a new user
GET /account/1 - returns a new user

-------- homes
GET /homes - returns the list of the homes
POST /homes - creates a new home

-------- hubs
GET /hubs - returns all hubs registered for this account
POST /hubs - registers a new hub (include encode / decode key)

-- hub sensor state batch
GET /hubs/1/all_sensors - returns all sensors with values
PUT /hubs/1/all_sensors - persists a snapshot of the devices with their sensors

-------- device manipulation
GET /hubs/{hubId}/devices - returns the list of devices including no sensor values
GET /hubs/{hubId}/devices/12 - returns the device with id 12
POST /hubs/{hubId}/devices - registers a new device (can be also done through a device/sensor state update)
PUT /hubs/{hubId}/devices/12 - updates the device with id 12

-------- sensor endpoints for one device
GET /hubs/{hubId}/devices/12/sensors - returns the list of sensors on this device along with their last measured values
POST /hubs/{hubId}/devices/12/sensors/0/value - updates the value of the sensor with index 0 on the device with id 12
